# АНК-ЮГ OS (тестовое задание)

Внутренняя операционная система для работы с **обращениями**: список с фильтрами, сортировкой и пагинацией, карточка с редактированием, **RBAC** (роли «Менеджер» / «Админ»), мок API через **MSW**.

## Стек

- React 18, TypeScript (strict), Vite 6  
- Tailwind CSS  
- TanStack Query, React Hook Form, Zod  
- MSW 2 (browser worker)

## Запуск

```bash
npm install
npm run dev
```

Откройте в браузере адрес, который выведет Vite (по умолчанию `http://localhost:5173`). В режиме разработки поднимается **Mock Service Worker**; данные обращений хранятся в памяти в рамках сессии.

Сборка и превью production:

```bash
npm run build
npm run preview
```

Линт:

```bash
npm run lint
```

## Архитектура (близко к Feature-Sliced Design)

| Слой | Содержимое |
|------|------------|
| `app` | `main.tsx`, `App.tsx`, провайдеры, маршрутизация, layout |
| `pages` | Страницы: список обращений, карточка обращения |
| `features` | Переключатель роли, модалка удаления |
| `entities` | `appeal`, `user` (типы, схемы Zod, API, query-хуки, auth) |
| `shared` | UI-kit, утилиты, MSW handlers |

Псевдоним импортов: `@/` → `src/`.

## Функциональность

- **Список**: таблица, фильтр по статусу, поиск по заголовку/описанию, сортировка по колонкам, пагинация; параметры отражаются в query string URL.  
- **Карточка**: просмотр и сохранение полей по схеме Zod; тосты об успехе/ошибке.  
- **RBAC**: удаление обращения доступно только в роли **Админ** (переключатель в шапке).  
- **Состояния**: загрузка (спиннер), пустой список, ошибка с кнопкой «Повторить».

## API (моки)

Базовый префикс: `/api/appeals`

- `GET /api/appeals` — список (query: `page`, `pageSize`, `sortBy`, `sortOrder`, `status`, `q`)  
- `GET /api/appeals/:id`  
- `PATCH /api/appeals/:id`  
- `DELETE /api/appeals/:id`  

После обновления версии `msw` в `package.json` при необходимости перегенерируйте worker:

```bash
npx msw init public --save
```

Файл `public/mockServiceWorker.js` должен соответствовать версии пакета `msw`.
