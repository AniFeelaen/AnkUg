# АНК-ЮГ OS (тестовое задание)

Внутренняя операционная система для работы с **обращениями**: список с фильтрами, сортировкой и пагинацией, карточка с редактированием, **RBAC** (роли «Менеджер» / «Админ»). **Бэкенда нет:** данные — **в памяти браузера** из `src/data/mockData.ts` (имитация задержки сети ~300 ms).

## Стек

- React 18, TypeScript (strict), Vite 6  
- Tailwind CSS  
- TanStack Query, React Hook Form, Zod  
- Демо-данные: `src/data` (без `fetch` к API)

## Запуск

```bash
npm install
npm run dev
```

Откройте в браузере адрес, который выведет Vite (по умолчанию `http://localhost:5173`).

Сборка и превью production:

```bash
npm run build
npm run preview
```

Линт:

```bash
npm run lint
```

## Архитектура (близко к Feature-Sliced Design)

| Слой | Содержимое |
|------|------------|
| `app` | `main.tsx`, `App.tsx`, провайдеры, маршрутизация, layout |
| `pages` | Страницы: список обращений, карточка обращения |
| `features` | Переключатель роли, модалка удаления |
| `entities` | `appeal`, `user` (типы, схемы Zod, API, query-хуки, auth) |
| `shared` | UI-kit, утилиты |
| `data` | Примеры обращений и in-memory store для демо |

Псевдоним импортов: `@/` → `src/`.

## Функциональность

- **Список**: таблица, фильтр по статусу, поиск по заголовку/описанию, сортировка по колонкам, пагинация; параметры отражаются в query string URL.  
- **Карточка**: просмотр и сохранение полей по схеме Zod; тосты об успехе/ошибке.  
- **RBAC**: удаление обращения доступно только в роли **Админ** (переключатель в шапке).  
- **Состояния**: загрузка (спиннер), пустой список, ошибка с кнопкой «Повторить».

## Демо-данные

- Список: `src/data/mockData.ts` (`mockAppeals`).  
- Логика списка / правок / удаления: `src/data/demoAppealsStore.ts` (состояние живёт в сессии вкладки).  
- Упрощённый хелпер: `src/data/mockClient.ts` → `getAppeals()`.

## Деплой на Vercel

Корень репозитория — **Root Directory** проекта. Сборка: `npm run build`, выходная папка: **`dist`**.  
В корне есть **`vercel.json`** с rewrite на `index.html`, чтобы при обновлении страницы на маршрутах вроде `/appeals/:id` не было 404.
